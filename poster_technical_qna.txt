廢片偵測器報告技術 Q&A 整理

一、Train 70% / Val 15% / Test 15% 是什麼意思？

Q：Train 70%、Val 15%、Test 15% 是什麼意思？

A：
這代表我們把全部 1,060 張照片分成三份，各自有不同用途。

以 1,060 張照片來說，大約會分成：
- Train 70%：約 742 張
- Val 15%：約 159 張
- Test 15%：約 159 張

Train 是訓練集，用來讓模型真正學習 Good 和 Bad 的差異。模型會看這些照片，根據預測錯誤不斷更新權重。

Val 是驗證集，不直接拿來更新權重，而是用來調整模型設定，例如 learning rate、訓練輪數、Bad 判定閾值，以及觀察模型是否過擬合。

Test 是測試集，會保留到最後才使用，用來評估模型對沒看過的新照片表現如何。因為模型訓練與調參時沒有使用 Test 資料，所以 Test Accuracy 比較能代表真實使用情境。

現場回答版本：
我們把 1,060 張照片分成 Train、Val、Test 三份。Train 70% 用來讓模型學習 Good 和 Bad 的差異；Val 15% 用來調整超參數和判定閾值；Test 15% 則完全保留到最後做獨立評估，確認模型對沒看過的照片也有一定判斷能力。這樣可以避免模型只是在訓練資料上表現好，而是比較接近真實使用情境。


二、MediaPipe、三層 Fallback、降低漏偵是什麼意思？

Q：MediaPipe 是什麼？

A：
MediaPipe 是 Google 提供的一套電腦視覺工具。在我們的專案裡，它的用途是從照片中偵測人臉的位置。

流程大致是：
原始照片 -> MediaPipe 偵測人臉 -> 裁切人臉 -> 輸入 MobileNetV3 判斷 Good 或 Bad。

如果沒有先偵測和裁切人臉，模型可能會看到太多背景、衣服、光線或場景資訊，反而比較難專注判斷表情。

Q：三層 Fallback 是什麼？

A：
Fallback 的意思是備援方案，也就是第一種方法失敗後，換另一種方法補救。

三層 Fallback 的意思是：
第一層找不到人臉 -> 換第二層
第二層還找不到 -> 換第三層

我們的三層大致是：
- 第一層 Short-range：適合近距離自拍、大臉。
- 第二層 Full-range：適合中距離、合照、臉比較小的照片。
- 第三層 Letterbox 自動裁切：適合有黑邊、比例特殊或人臉位置不標準的照片。

Q：降低漏偵是什麼？

A：
漏偵的意思是照片裡明明有人臉，但系統沒有偵測到。如果人臉沒有被抓出來，後面的模型就沒辦法判斷表情好不好。

所以降低漏偵就是透過三層 Fallback，盡量把照片裡的人臉成功找出來，減少被誤判為無人臉的情況。

現場回答版本：
MediaPipe 是我們用來做人臉偵測的工具。因為照片可能有近拍、遠拍、合照或特殊構圖，如果只用一種偵測方式，容易漏掉部分人臉。所以我們設計三層 Fallback：第一層偵測失敗時換第二層，第二層失敗再換第三層。這樣可以降低漏偵，也就是減少照片裡有臉但系統沒抓到的情況，讓後面的表情分類模型有更穩定的輸入。


三、為什麼要特別選 224 x 224？

Q：為什麼人臉裁切後要統一成 224 x 224？

A：
主要有三個原因。

第一，MobileNetV3-Large 這類 ImageNet 預訓練模型常用的輸入尺寸就是 224 x 224。使用這個尺寸可以比較完整地沿用預訓練模型原本學到的特徵。

第二，模型輸入需要固定尺寸。原始照片可能有直式、橫式、不同解析度，如果不先統一大小，模型很難直接批次訓練與推論。

第三，224 x 224 在細節與速度之間取得平衡。如果尺寸太小，眼睛和嘴角等表情細節可能消失；如果尺寸太大，訓練和推論會變慢，也更吃記憶體。

現場回答版本：
我們選 224 x 224 是因為 MobileNetV3-Large 這類預訓練模型常用這個輸入尺寸。把裁切後的人臉統一成 224 x 224，可以符合模型架構，也方便批次訓練。同時這個大小在表情細節和推論速度之間取得平衡，不會太小導致眼睛、嘴角資訊消失，也不會太大造成運算成本過高。


四、MobileNetV3-Large、SE Attention、輸出 Bad 信心分數是什麼意思？

Q：MobileNetV3-Large 是什麼？

A：
MobileNetV3-Large 是一種 CNN 深度學習影像分類模型。它的工作是看一張裁切後的人臉圖片，判斷它比較像 Good 還是 Bad。

流程是：
原始照片 -> 偵測人臉 -> 裁切成 224 x 224 -> 輸入 MobileNetV3-Large -> 判斷 Good 或 Bad。

MobileNet 系列原本就是為低運算資源環境設計的模型，速度快、模型相對輕量。Large 代表它是 MobileNetV3 中容量比較大的版本，通常準確率會比 Small 版好。

Q：SE Attention 是什麼？

A：
SE Attention 全名是 Squeeze-and-Excitation Attention，是一種注意力機制。它可以讓模型自動判斷哪些特徵比較重要，哪些比較不重要。

在我們的任務中，重要特徵可能包含：
- 眼睛有沒有閉起來
- 嘴角有沒有歪
- 臉部表情是否自然
- 五官區域是否清楚

比較不重要的可能是背景、衣服、頭髮邊緣或光線顏色。SE Attention 可以幫助模型更聚焦在與表情品質有關的特徵。

Q：輸出 Bad 信心分數是什麼？

A：
模型最後不是只單純說 Good 或 Bad，而是會輸出一個分數。這個分數可以理解為模型認為這張照片是 Bad 的信心程度。

例如：
- Bad 信心分數 0.92：模型很確定這張是 Bad。
- Bad 信心分數 0.67：模型偏向判斷為 Bad。
- Bad 信心分數 0.48：模型不太確定，接近邊界。
- Bad 信心分數 0.12：模型認為它不像 Bad，比較像 Good。

系統可以設定一個閾值，例如 0.5：
Bad 信心分數 >= 0.5 -> 判定為 Bad
Bad 信心分數 < 0.5 -> 判定為 Good

現場回答版本：
MobileNetV3-Large 是我們用來做人臉表情分類的 CNN 模型，它會接收 224 x 224 的人臉圖片，判斷這張臉比較像 Good 還是 Bad。SE Attention 是模型中的注意力機制，可以強化像眼睛、嘴角這類和表情品質有關的特徵。最後模型會輸出 Bad 信心分數，也就是它認為這張照片是廢片的程度，系統再根據這個分數和閾值決定要不要列入建議刪除。


五、三層 Fallback 人臉偵測架構是什麼意思？

Q：Short-range、Full-range、Letterbox 這三層是什麼？

A：
這一塊是在說系統不是只用一種方式找人臉，而是設計了三層備援流程。

第一層 Short-range：
專門處理近距離大臉，例如自拍照、臉佔畫面很大的照片。它對近距離人臉比較準，但如果是合照、遠距離、臉比較小，就可能偵測不到。海報上寫漏失基線 27 張，意思是如果只靠第一層，原本有 27 張照片裡的人臉會被漏掉。

第二層 Full-range：
用來補足中遠距離的人臉，例如合照、半身照、臉比較小的照片。加入第二層後，原本漏掉的照片被救回一部分，漏失從 27 張降到 8 張。

第三層 Letterbox：
針對更特殊的情況，例如照片有黑邊、截圖比例怪、人臉位置太偏或原始構圖不標準。Letterbox 自動裁切會先對圖片做比例調整或裁切，讓人臉偵測器更容易找到臉。加入第三層後，漏失從 8 張降到 3 張。

現場回答版本：
我們設計三層 Fallback 人臉偵測，是因為照片可能有近拍、遠拍、合照或截圖黑邊。如果只用一種偵測方式，容易漏掉人臉。第一層 short-range 處理近距離自拍，第二層 full-range 補足中遠距離合照，第三層 letterbox 處理特殊比例或黑邊構圖。最後漏偵從 27 張降到 3 張，代表前處理穩定度有明顯提升。


六、兩階段微調與注意力機制是什麼意思？

Q：Stage 1 凍結是什麼意思？

A：
凍結的意思是先把模型前面大部分已經學好的特徵固定住，不讓它更新，只訓練最後的分類頭。

MobileNetV3 原本已經在大型影像資料集上學過很多基本視覺特徵，例如邊緣、紋理、形狀和臉部輪廓。因為我們的資料集只有 1,060 張，如果一開始就讓整個模型全部更新，容易過擬合。

所以第一階段先凍結底層，只訓練最後負責判斷 Good / Bad 的分類層。Learning Rate 1e-3 代表第一階段學習率比較大一點，讓新的分類頭能比較快學會任務。

Q：Stage 2 解凍是什麼意思？

A：
解凍的意思是開放模型後面一小部分權重，讓它針對我們的資料再微調。

海報寫解凍最後 3 個 Inverted Residual Blocks，意思是只調整 MobileNetV3 後面比較高階的特徵層，而不是整個模型全部重訓。

後面的層通常負責比較抽象的特徵，例如表情、五官狀態、臉部品質；前面的層則多半是邊緣、顏色、紋理這些通用特徵。

第二階段 Learning Rate 1e-5 比第一階段小很多，因為這時候是在微調已經訓練好的模型，不能改太大，避免把原本學好的特徵破壞掉。

Q：右邊的人臉注意力圖是什麼意思？

A：
右邊的人臉圖是在表達 SE Attention 會幫助模型更關注眼睛與嘴角，因為這些區域最能反映表情是否崩壞，例如閉眼、嘴歪或表情不自然。

現場回答版本：
這一塊是在說我們的模型訓練方法。我們使用 MobileNetV3-Large 的預訓練權重，不是從零開始訓練。第一階段先凍結底層特徵，只訓練最後的分類頭，讓模型先學會 Good / Bad 任務。第二階段再解凍最後 3 個 Inverted Residual Blocks，用較小的 learning rate 微調高階特徵，讓模型更適應表情判斷。同時 SE Attention 會幫助模型強化眼睛、嘴角這些跟表情品質相關的特徵。


七、前後端分離的本機 Web 架構是什麼意思？

Q：前後端分離是什麼意思？

A：
前後端分離的意思是畫面介面和資料處理分開做。

我們的系統分成兩部分：
- 後端 Backend：FastAPI / Python，負責 AI 推論、掃描照片、移到 Trash、還原照片。
- 前端 Frontend：Vue 3 / JavaScript，負責顯示網頁介面、按鈕、卡片、進度條、掃描結果。

簡單說，前端負責使用者看到和操作的畫面，後端負責真正處理照片和跑 AI。

Q：完全離線運算，照片不上傳雲端是什麼意思？

A：
意思是使用者的照片都留在自己的電腦裡，不會傳到 Google、雲端伺服器或外部 API。後端 FastAPI 是跑在本機，例如 http://localhost:8000，所以 AI 推論、掃描資料夾、刪除與還原都在本機完成。

這樣的好處是：
- 隱私比較安全
- 不需要把私人照片上傳
- 沒網路也可以運作
- 使用者對照片保有控制權

Q：REST API 是什麼？

A：
REST API 是前端和後端溝通的方式。前端不能直接操作模型或檔案系統，所以會透過 API 跟後端說：
- 我要開始掃描
- 我要看掃描進度
- 我要取得分析結果
- 我要把這些照片移到 Trash
- 我要還原照片

後端收到請求後，才真的去執行動作。可以把 REST API 想成前端和後端之間的溝通橋樑。

Q：FastAPI 後端在做什麼？

A：
後端主要處理比較核心、比較危險或需要 Python 的工作，例如讀取本機照片、用 MediaPipe 偵測人臉、用 MobileNetV3 判斷 Good / Bad、儲存掃描結果、把照片移到 Trash、從 Trash 還原照片。

Q：Vue 3 前端在做什麼？

A：
前端負責互動體驗，例如顯示掃描進度、顯示 Good / Bad / 未偵測人臉結果、用卡片方式排列照片、讓使用者勾選照片、按下刪除或還原按鈕，以及不用重新整理頁面就更新畫面。

非同步掃描輪詢的意思是後端掃描照片需要時間，所以前端會每隔一段時間問後端：「現在掃到幾張了？」然後更新進度條。

現場回答版本：
這一塊是在說我們的系統採用前後端分離架構。後端使用 Python FastAPI，負責 AI 推論、照片掃描、軟刪除和還原；前端使用 Vue 3，負責使用者看到的 Web UI，例如掃描進度、照片卡片和勾選操作。前端和後端透過 REST API 溝通。整個系統都跑在本機端，所以照片不需要上傳雲端，可以降低隱私外流風險。
