01 專案背景 Background

數位攝影時代,每人每天產出大量照片,其中夾雜閉眼、嘴歪、模糊等廢片。既有相簿軟體僅能做品質粗篩(模糊、過曝偵測),無法判斷「表情品質」這個高階語意。本研究以遷移學習為核心,自訓深度學習模型自動判別表情自然與崩壞,並開發 FastAPI + Vue 3 Web UI 讓使用者本機運行、完全離線地批次清理。核心挑戰:在小資料集(< 1,500 張)條件下穩定泛化到從未見過的人臉與場景。

既有產品提供的自動篩選核心局限
iCloud / Google Photos智慧分類、面孔識別、相似照片群組不會主動建議刪除,使用者仍需逐張決定
Gemini Photos找出重複、模糊、相似照片僅基於像素相似度與品質指標,不理解表情
CleanMyMac / Photo Sweeper大量、模糊、低解析度偵測同上,無法分辨「光線好但表情崩壞」的照片
Apple 最佳照片建議連拍中推薦最佳一張黑盒、不可調、無法批次處理

04 系統特色 Features
完全離線、保護隱私
所有運算在本機 PyTorch + MediaPipe 完成,照片不會上傳任何雲端。對家庭、私密照片等敏感資料尤其重要。
自訓深度學習模型
使用 PyTorch 從 ImageNet 預訓 backbone 開始,在自建資料集上完整訓練,非套件包裝。
三層 Fallback 偵測
創新整合 short-range + full-range + letterbox 三層偵測,漏失率僅 0.3%,連 iPhone 截圖都能處理。
軟刪除與還原機制
所有刪除動作搬到 Trash 並寫入 manifest.json,可隨時還原,徹底避免誤刪悲劇。
Apple Silicon 加速
自動偵測 MPS / CUDA / CPU 後端,在 M2 Mac 上能快速完成批次掃描,無需雲端 GPU。
透明的信心分布
UI 內建信心 histogram 視覺化,讓使用者理解模型的判斷依據,而非接收黑盒結果。
05 系統技術 Technology

模型架構與訓練

  • Backbone:MobileNetV3-Large(ImageNet 預訓),內建 SE Attention 模組
  • Stage-1:凍結 features,訓練 2 類 head(lr=1e-3 · 15 ep)
  • Stage-2:解凍最後 3 個 inverted residual blocks(lr=1e-5 · 5 ep)
  • Optimizer:Adam + Weight Decay 1e-4
  • Loss:CrossEntropy + Label Smoothing 0.05
  • Augmentation:RandomResizedCrop · ColorJitter · RandomRotation · RandomErasing

人臉偵測創新

  • Primary:MediaPipe short-range(專責肖像、自拍)
  • Fallback:MediaPipe full-range(專責合照、中遠距人物)
  • Letterbox 自動裁切:偵測純色邊框(std < 8)並裁掉,解決 iPhone 螢幕截圖
  • bbox 向外擴張 15% 保留完整頭部,統一 resize 至 224×224
  • min_face_px = 20, min_confidence = 0.3

推論優化

  • TTA(Test-Time Augmentation):原圖 + 水平翻轉,機率平均提升穩定性
  • 決策閾值自動掃描:範圍 0.30~0.70,取代死板的 0.5(本研究最佳為 0.47)
  • Lazy Singleton 快取:模型 / detector 只在首次呼叫時載入
  • 自訂例外:NoFaceDetectedError,UI 端 try/except 即可跳過
  • 支援 BGR ndarray / Path / str 多種輸入

嚴謹評估

  • Train / Val / Test 三段切分:70 / 15 / 15(SEED=42 可重現)
  • Confusion Matrix + Per-class P/R/F1:用 sklearn.metrics
  • Top-N 錯誤案例分析:挑出「最自信卻判錯」的樣本
  • 獨立 evaluate.py 腳本:不重訓即可重新評估任何 checkpoint
  • 9 個 pytest 單元測試:覆蓋推論流程的邊界條件

前後端分離架構

  • 後端:FastAPI(Python)提供 REST API,自動 OpenAPI 文件
  • 前端:Vue 3 + Vite,SPA 架構,reactive 細粒度更新
  • 掃描非同步化:背景 thread 執行推論,前端輪詢進度(500ms)
  • 同源部署:vite build → FastAPI 同時服務 API 與靜態檔
  • 本機運行:uvicorn :8000 + Vite :5173,無雲端依賴

UX 效能最佳化

  • Server-side 縮圖快取:Pillow 即時 resize + MD5 hash 快取(backend/cache/)
  • 瀏覽器 lazy loading:<img loading="lazy">,捲到才下載
  • HTTP cache headers:相同 URL 由瀏覽器原生快取,翻頁無延遲
  • Vue reactive grid:勾選 checkbox 只更新該卡片 DOM(非整頁重渲染)
  • 分頁渲染:預設每頁 30 張,大型資料集不卡頓
